Java位运算

发表于 2020-08-06 | 分类于 Java基础

基本概念

在学习位运算之前先了解下机器数,真值,原码,反码,补码的概念

机器数

一个数在计算机中的二进制表现形式，叫做这个数的机器数

机器数是带符号的，在计算机中一个数的最高位存放符号，正数为0，负数为1

比如，十进制中的数3，计算机字长为8位，转换为二进制就是0000 0011

十进制中的数-3,为1000 0011。

真值

上面我们有说道，计算机中一个数的最高位是存放符号的，1000 0011，其中最高位1代表是负数，其真正数值为-3，而不是131（1000 0011，转换为十进制其实是131）,所以为了区别，带符号的机器数对应的真正数值称为机器数的真值

比如0000 0011的真值=+000 0011=3;1000 0011的真值=-000 0011=-3

原码

原码就是符号位加上真值的绝对值，即用第一位表示符号，其余表示值。

也就是一个数取其绝对值，转换为二进制数，首位表示符号位，就是原码

正数的原码是本身，负数的原码是其正数的原码，将符号位改为1

比如：1的原码：[+1]=0000 0001；-1的原码：[-1]=1000 0001

反码

正数的反码是其本身

负数的反码是其原码的基础上，符号位不变，其余取反

负数的反码一般无法直观看出，都需要先将其转换为原码再计算

1的原码：[+1]=0000 0001;1的反码：[+1]=0000 0001

-1的原码：[-1]=1000 0001;-1的反码：[-1]=1111 1110

补码

正数的补码是其本身

负数的补码是在其原码的基础上，符号位不变，其余取反，最后+1

也就是在反码的基础上+1

1的原码：[+1]=0000 0001;1的反码：[+1]=0000 0001;1的补码：[+1]=0000 0001;

-1的原码：[-1]=1000 0001;-1的反码：[-1]=1111 1110;-1的补码：[-1]=1111 1111

为什么要使用原码,反码,补码

通过上面我们可以知道，反码，补码都不利于人脑的直接识别，那为什么不直接用源码进行计算？

对于[-1]=1000 0001，我们知道，最高位为符号位，但计算机不知道呀，想让它知道，那会使它的基础电路设计的非常复杂。我们知道，减去一个数，就等于加一个负数，那减法也可以给它省了。

首先来看原码

计算十进制表达式：1-1=0

1-1=1+（-1）=[0000 0001]原+[1000 0001]原=[1000 0010]原=-2

这显然是不正确的，所以这也是计算机内部不使用原码表示一个数的原因

为了解决原码做减法的问题，出现了反码

1-1=1+(-1)=[0000 0001]原+[1000 0001]]原=[0000 0001]反+[1111 1110]反=[1111 1111]反=[1000 0000]=-0

发现用反码计算减法，结果的真值部分是正确的，但“0”这个特殊的值有问题，虽说咱们知道-0和0是一样的，但计算机不知道，它会出现[0000 0000]原,[1000 0000]原俩个编码来表示0.

于是补码的出现，解决了这个问题，也解决了多编码代表1个值的问题。

1-1=1+（-1）=[0000 0001]原+[1000 0001]原=[0000 0001]补+[1111 1111]补=[0000 0000]补=[0000 0000]原

上学学的都还给老师了，理解这个查了半天资料 -_-

位运算

计算机中所有数据都是以补码的形式存在的

负数先转换为补码在进行运算

&：与运算

如果俩个二进制位都是1，那么运算结果是1，其余都是0

例如：1+2进行& 与运算

例如：1+2进行| 或运算

~：非运算【取反运算】

字面意思，取反

简单算法：取相反数-1：例如：~0=-1;~5=-6；~-5=4

^：异或运算

如果俩个二进制位相同，为0，不同为1

例如：1+2进行^ 或运算
